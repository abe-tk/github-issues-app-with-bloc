## ADDED Requirements

### Requirement: コメント一覧の表示
Issue詳細画面においてそのIssueに紐づくコメントを一覧表示しなければならない（SHALL）。各コメントはbody、作成者、作成日時を表示する。

#### Scenario: コメントがある場合
- **WHEN** ユーザーがコメント付きのIssue詳細画面を表示する
- **THEN** コメントが作成日時順に一覧表示される
- **THEN** 各コメントにbody、作成者名、作成日時が表示される

#### Scenario: コメントがない場合
- **WHEN** ユーザーがコメントのないIssue詳細画面を表示する
- **THEN** 「コメントはありません」のメッセージが表示される

#### Scenario: コメント読み込み中
- **WHEN** コメントをAPIから取得中
- **THEN** ローディングインジケーターが表示される

### Requirement: コメントの投稿
ユーザーはIssue詳細画面から新しいコメントを投稿できなければならない（MUST）。

#### Scenario: コメントを投稿する
- **WHEN** ユーザーがコメント入力欄にテキストを入力し、送信ボタンをタップする
- **THEN** GitHub APIにコメントが投稿される
- **THEN** コメント一覧に投稿したコメントが追加表示される

#### Scenario: 空のコメントは投稿できない
- **WHEN** ユーザーがコメント入力欄を空のまま送信ボタンをタップする
- **THEN** 送信は実行されない

#### Scenario: コメント投稿中のフィードバック
- **WHEN** コメント投稿APIを呼び出し中
- **THEN** ローディングインジケーターが表示され、送信ボタンは無効化される

#### Scenario: コメント投稿失敗時
- **WHEN** コメント投稿APIがエラーを返す
- **THEN** エラーメッセージが表示され、入力内容は保持される
